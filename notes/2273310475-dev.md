Reactive Network Study Notes

Topic: Basic Concepts of Ethereum, Smart Contracts, Transactions, and Events

1\. What is ETH

ETH (Ether) is the native currency of the Ethereum blockchain.

It is used for:

\- Sending value between accounts

\- Paying gas fees for transactions

\- Interacting with smart contracts

Summary:

ETH = the money used inside the Ethereum network.

2\. What is Ethereum

Ethereum is a blockchain network that allows users to run programs on-chain.

Those programs are called smart contracts.

Summary:

Ethereum = a decentralized computer network that can run programs.

3\. What is a Smart Contract

A smart contract is a program stored and executed on the blockchain.

It automatically runs predefined rules when triggered by a transaction.

Example:

If a user sends ETH to a contract,

the contract can execute code automatically.

Summary:

Smart Contract = program on the blockchain.

4\. What is a Transaction

A transaction is an action initiated by a user.

Examples:

\- Sending ETH to another wallet

\- Calling a function in a smart contract

Structure of a transaction usually includes:

\- sender

\- receiver

\- value (ETH)

\- data

\- gas

Summary:

Transaction = a user action sent to the blockchain.

5\. What is an Event

An event is a log recorded by a smart contract during execution.

In Solidity, events are defined like this:

event Received(address sender, uint value);

When the contract runs:

emit Received(msg.sender, msg.value);

The blockchain records this event.

Summary:

Event = a log generated by contract execution.

6\. Relationship Between Transaction and Event

Transaction triggers contract execution,

and contract execution can emit events.

Flow:

Transaction

↓

Contract Execution

↓

Event Log

Summary:

Transaction = action

Event = record/log of what happened.

7\. What the Demo Contract Does

In the Reactive Network demo:

User sends ETH to the contract.

The contract:

1\. Receives ETH

2\. Emits a "Received" event

3\. Sends the ETH back to the sender

Simplified logic:

User sends ETH

↓

Contract receives ETH

↓

emit Received event

↓

ETH returned to sender

Purpose:

The goal is not transferring money.

The goal is generating an event for Reactive Network to monitor.

8\. Why Events are Important

Many blockchain systems monitor events instead of transactions.

Examples:

\- DeFi bots

\- Indexers

\- Oracles

\- Reactive Network

Event-driven systems can automatically respond when something happens.

Example:

Event occurs

↓

Reactive system detects it

↓

Automatic contract execution

9\. Core Idea of the Demo

This contract acts as an event generator.

Its role is to produce an event when ETH is sent to it.

Reactive Network can then monitor this event and trigger further actions.

10\. Key Takeaways

ETH = money on Ethereum

Smart Contract = program on blockchain

Transaction = user action

Event = log emitted by a contract

Relationship:

Transaction → Contract Execution → Event
