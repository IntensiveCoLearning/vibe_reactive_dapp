1\. Reactive Smart Contracts (RSC)

Reactive Smart Contracts are smart contracts that are triggered by events instead of user transactions.

In traditional blockchains, smart contracts execute only when a user sends a transaction.

In Reactive Network, smart contracts can execute automatically when specific events occur.

Comparison:

Traditional Smart Contract

```
User → Transaction → Smart Contract Execution
```

Reactive Smart Contract

```
Event → Automatic Execution
```

Key idea:

Reactive Smart Contracts follow an event-driven model.

2\. Core Logic of Reactive Network

The most important concept in Reactive Network is the event-driven workflow.

Basic flow:

```
Event → Listen → Execute → Action
```

Explanation:

1\. An event occurs on a blockchain

2\. Reactive Network listens to the event

3\. ReactVM executes the reactive smart contract

4\. The contract performs the required action

Simplified model:

```
Event → Reaction
```

3\. Problems Reactive Network Solves

Reactive Network enables automation in blockchain systems.

Traditional blockchains rely heavily on users or bots to trigger transactions.

Reactive Network allows contracts to respond automatically to blockchain events.

Key use cases include:

\- Automated DeFi trading

\- Automatic liquidation mechanisms

\- DAO governance automation

\- Cross-chain interactions

\- Event-triggered smart contract execution

4\. Phase 1 Summary

Traditional blockchains are transaction-driven systems.

Smart contracts run only when a user sends a transaction.

Reactive Network introduces an event-driven layer that listens to blockchain events

and automatically triggers smart contract execution.

Core idea:

```
Event → Listen → Execute → Action
```

Reactive Network improves automation and reduces reliance on external bots.

**(1)理解 Reactive Network 与传统 EVM 的区别**

**A:「 EVM = Ethereum Virtual Machine」**

| 类比 | 对应 |
| --- | --- |
| 电脑操作系统 | EVM |
| 程序 | 智能合约 |

`智能合约代码（） ↓ 在 EVM 里运行（运行智能合约的环境。） ↓ 产生结果`

**B:传统EVM：被动系统** **Traditional Blockchain-**smart contrac 不能自己运行 需要有人call—— run

`用户调用（用户发送交易） ↓ 智能合约执行（EVM环境下） ↓ 产生结果（区块链更新状态）`

eg：Ethereum 如有user/bot 想要买 执行合约buy() 那么 他要先发送交易call buy() 再执行合约

`User → Transaction → Smart Contract`

so:很多DeFi项目用bot:自动清算、交易、套利

**C:Reactive**

`event → ReactVM → smart contract`

| 系统 | 执行环境 |
| --- | --- |
| 传统区块链 | EVM |
| Reactive Network | ReactVM |

`事件发生 ↓ Reactive Network 监听到 ↓ ReactVM 开始执行代码 ↓ 智能合约运行`

（2）ReactVM + Reactive Network

**Reactive Network listens to events, and ReactVM executes smart contracts.**

`Event happens ↓ Reactive Network detects it ↓ ReactVM executes the contract`

| 对比项 | 传统 EVM | Reactive |
| --- | --- | --- |

| 怎么开始执行 | 需要有人调用 | 有事件就自动执行 |

| 谁来触发 | 用户 / 机器人 bot | 系统根据事件触发 |

| 执行方式 | 手动 | 自动 |

| 核心逻辑 | 先调用，再执行 | 先发生事件，再执行 |

| 简单理解 | 点一下——它才动 | 条件到了————自己动 |

| 例子 | 用户调用合约买币 | 价格跌到条件，自动买币 |

| 类型 | 最简单理解 |
| --- | --- |
| 传统 EVM | 人调用，合约才执行 |
| Reactive | 事件发生，合约自动执行 |

可以理解为一种 **自动化**，而且是**事件驱动的自动化**。

| 场景 | 发生什么 |
| --- | --- |
| 门铃 | 有人按门铃 → 门铃响 |
| 微信 | 收到消息 → 手机提醒 |
| Reactive程序 | 价格下降 → 自动执行交易 |
