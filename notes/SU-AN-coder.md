# Reactive 合约的有状态性（Stateful）与历史数据累积

Reactive 合约一个关键技术特性是**有状态（Stateful）**，这让它远强于一次性触发的简单自动化逻辑。合约不仅能响应单次事件，还可以在自身状态中**持续存储、累积历史数据**，比如多笔交易的价格记录、多轮预言机更新值、累计交易量、时间序列数据等。在触发逻辑时，RC 不是只看当前事件，而是会结合**历史状态 + 最新事件**共同判断是否执行。例如可以根据一段时间内的价格均值、多次 oracle 数据的平均值、连续几次的流动性变化来决策，而不是单次触发就动作。这种能力让复杂的时序策略、动态阈值判断、多数据源聚合等场景，都能在完全去中心化、无外部数据库的情况下纯链上实现，也是它比普通链下机器人更安全可信的核心原因之一。

# 睿应式合约内部完整运行

Reactive 合约在内部运行时，遵循一套完整且有状态的事件驱动流程，与传统合约单次调用单次执行的模式完全不同。开发者在编写合约时，首先需要明确指定要监听的目标链、合约地址以及事件 Topic0，合约会持续监听这些链上行为，包括转账、Swap、清算、投票、预言机更新等各类合约事件。当目标事件被捕获后，Reactive Network 会自动触发合约内部逻辑，并基于事件数据进行计算。由于 RCs 是有状态（stateful）的，它可以在自身状态中累积历史数据，结合最新事件综合判断是否满足执行条件，而不是只依据单次信息做决策。判断完成后，合约会先更新自身内部状态，保持数据持续同步，随后在满足条件时向目标 EVM 链发起自动交易执行。整个流程完全在链上可信运行，不需要外部 bot、不需要私钥托管，也不需要用户签名触发，实现了从事件监听、状态累积、逻辑判断到自动执行的全闭环去中心化自动化。

# Reactive 合约的状态性

Reactive 合约（RCs）与传统自动化方案（如链下机器人）的一个关键技术区别，在于它是**原生有状态（Stateful）** 的，且状态与事件驱动逻辑深度绑定。

传统链下机器人若要累积历史数据（如过去 1 小时的价格均值、累计交易量），需依赖外部数据库存储，存在数据一致性与安全性风险；而 RCs 的状态直接存储在 Reactive Network 上，开发者可在合约中定义状态变量来**累积、更新历史事件数据**（如多次 Oracle 报价的平均值、多笔 Swap 的价格轨迹）。

其技术执行流程非常清晰：首先，开发者在合约中通过指定「目标链 ID + 目标合约地址 + 事件 Topic 0」完成订阅；当 Reactive Network 监听到目标事件后，会先将事件数据传入合约，**更新合约内部状态**（如累加价格、记录时间戳）；随后，合约可基于「历史状态 + 新事件数据」进行复合判断（如 “过去 3 次 Oracle 报价均值跌破 2000 且当前最新价格触发阈值”）；若条件满足，再通过 Reactive Network 的跨链调度能力，在目标 EVM 链上异步发起执行交易。

这种 “状态存储在链、事件触发更新、逻辑基于历史 + 实时” 的设计，既保证了数据的去中心化可信，又让复杂的时序策略（如移动平均线止盈、累计交易量阈值触发）无需依赖链下组件即可纯链上实现。

# Reactive 中的订阅与触发机制

Reactive 合约的运行依赖**事件订阅 + 条件触发**这套核心机制。开发者在合约中不是写 “立刻执行” 的逻辑，而是先声明：要监听哪条链、哪个合约、哪一类事件（如 Swap、Transfer、清算等），并设定触发条件。

Reactive Network 会持续监听这些链上事件，当目标事件发生并匹配预设条件时，网络会主动调用合约里对应的响应函数，在 ReactVM 中完成计算，并将结果异步提交到目标链执行。

整个过程不需要用户签名、不需要外部调用，实现了真正的**链上自治触发**，这也是它和传统合约 “调用即执行” 模式最关键的区别。

# Reactive 对 DeFi 开发者的价值

对 DeFi 开发者来说，Reactive 最大的价值是**把 “链上自动化” 变成原生能力**。

以前要做自动止盈、自动清算、跨链策略、动态费率等功能，必须自己开发后端机器人、对接预言机、管理私钥、处理高可用与安全问题，开发成本高、运维复杂、安全性差。

现在用 Reactive，开发者只需要在合约里定义**监听哪些事件、满足什么条件、执行什么逻辑**，剩下的事件监听、触发调度、跨链执行都由网络原生处理。不需要中心化服务，不需要托管，不需要额外运维，就能写出真正去中心化、无人值守的自动化协议。

这极大降低了 DeFi 创新门槛，让开发者专注业务逻辑，而不是纠结于机器人和运维

# Reactive 如何实现多链事件监听

Reactive 并不是一条单独的公链，而是一个**多链事件感知与执行协调层**，它的核心能力之一，就是可以同时监听多条 EVM 链上的事件。

传统智能合约只能看到自己所在链的数据，想获取其他链的信息必须依赖跨链桥或预言机，延迟高、结构复杂。而 Reactive Network 会直接同步多条公链的区块与事件，睿应式合约可以直接订阅并监听**以太坊、Base、Arbitrum、Polygon**等不同链上的合约行为，比如 Swap、清算、转账、Mint 等。

一旦目标链上出现符合条件的事件，合约会立刻在 ReactVM 中处理逻辑，并把结果异步执行到目标链上，实现 “**监听 A 链 → 计算在 ReactVM → 执行在 B 链**” 的完整闭环。这种原生多链感知能力，让跨链自动化、多链策略、全局清算等场景第一次真正去中心化、低成本实现。

# 控制反转（IoC）在 Reactive 里的意义

Reactive 里一个非常关键的设计思想，就是**控制反转（IoC）**，这也是它和传统 EVM 合约最底层的思维区别。

传统智能合约是**用户 / 外部系统控制流程**：必须由人或机器人主动发起交易、调用函数，合约才会执行一步。整个执行的 “控制权” 在合约外面，合约本身只是被动等待。

而 Reactive 把控制权反转了过来：**合约自己掌握执行时机**。

你只在部署时定义好：监听什么事件、满足什么条件、执行什么逻辑。之后整个流程由网络和合约自主驱动 —— 事件来了 → 条件满足 → 自动执行，不需要外部再去触发。

简单理解：

-   传统合约：**你喊它，它才动**
    
-   Reactive 合约：**它盯着，自己动**
    

控制反转让智能合约第一次真正具备**长期、自治、无人值守**的能力，为 DeFi 全自动策略、链上机器人、跨链自动化提供了最底层的范式支撑。

# 事件驱动睿应式智能合约

睿应式智能合约（Reactive Smart Contracts）是基于**事件驱动**模型设计的新一代智能合约，也是 Reactive Network 最核心的创新。与传统 EVM 合约必须由外部账户主动发起交易才能执行不同，它具备原生的**事件监听、条件判断、自动执行**能力。

合约可以预先订阅并监听多条 EVM 链上的任意事件，例如 DEX 中的 Swap、价格变动、借贷协议的清算、跨链桥的存款 / 提款、DAO 投票等。当监听到的事件满足预设条件时，合约会自动在 ReactVM 中完成逻辑计算，并将结果异步执行到目标链上，全程不需要用户签名、不需要中心化机器人、也不需要托管私钥。

这种设计从根本上解决了传统合约 “被动、依赖外部触发、自动化不安全” 的痛点，让链上策略真正实现**去中心化、无托管、7×24 小时全自动运行**，特别适合止盈止损、自动做市再平衡、跨链套利、清算机器人等 DeFi 场景。

# Reactive 核心架构（ReactVM + Reactive Network）

Reactive 并非新公链，而是专为多链 EVM 生态打造的区块链协调层，其核心架构由 ReactVM 和 Reactive Network 两部分组成，二者协同工作，支撑起睿应式智能合约的自主运行。Reactive Network 作为多链协调中枢，主要负责实时同步所有 EVM 兼容链的链上事件，管理睿应式合约的事件订阅、触发与执行，同时将最终执行结果无缝结算到任意目标链，解决了多链数据碎片化、跨链操作割裂的核心痛点，实现全链事件的统一监控与跨链协同。

ReactVM 则是 Reactive 专属的并行化 EVM，专门承接睿应式合约中的重计算逻辑，比如多链价格聚合、复杂 DeFi 策略运算等。不同于传统 EVM 需在昂贵的 L1 链上执行所有计算，ReactVM 可实现并行化计算，既大幅降低了链上 Gas 消耗，又提升了合约执行效率，避免了重计算导致的交易延迟。

二者相辅相成，Reactive Network 负责 “监听与调度”，ReactVM 负责 “高效计算”，共同构成了 Reactive 网络的底层支撑，让睿应式合约的 “事件驱动、自主执行、全链兼容” 成为可能，也为多链自动化场景（如跨链套利、全链策略执行）提供了高效、低成本的解决方案。

# 传统 EVM 合约与睿应式合约（RCs）的差异

传统 EVM 智能合约的核心痛点的是 “被动执行”，本质是需要外部触发（用户签名交易、中心化机器人调用）才能运行，自身不具备任何事件监听和自主执行能力。比如想要实现 Uniswap 价格止盈止损，只能依赖链下机器人盯盘、托管私钥发起交易，不仅存在私钥泄露、单点故障的风险，还需要额外支付机器人运维成本，且自动化流程不够透明可信。

而 Reactive 网络中的睿应式合约（Reactive Smart Contracts，RCs），彻底打破了这种被动模式，核心是 “事件驱动 + 自主执行”。它依托 Reactive Network 的多链事件同步能力和 ReactVM 并行计算支撑，可实时监听多条 EVM 链的指定事件（如 DEX 交易事件、借贷清算事件、跨链转账事件），用户只需提前预设 “触发条件 + 执行逻辑”，合约就会在条件满足时，自动在目标链上执行操作，全程无需人工干预、无需托管私钥、无中心化中介参与。

这种 “控制反转（IoC）” 的设计，是睿应式合约与传统合约的根本区别 —— 它让智能合约从 “等待调用的工具”，变成了 “能自主感知、主动响应的去中心化程序”，既解决了传统合约自动化依赖链下中介的痛点，又降低了链上自动化的操作门槛和 Gas 成本，尤其适合 DeFi 策略自动执行、跨链操作、全链流程自动化等场景，也是 Reactive 思维的核心体现。
